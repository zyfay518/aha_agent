# 页面与用户流程规格

> 版本：0.1  
> 状态：阶段 0 验收候选  
> 原则：移动端优先、最少输入、AI 预填、用户确认

## 1. 信息架构

MVP 网页包含以下入口：

```text
公开区域
├── 邀请落地页 /invite/:code
├── 登录 /login
└── 注册 /signup

登录区域
├── 衣橱首页 /wardrobe
├── 添加单品 /wardrobe/new
├── 确认识别 /wardrobe/new/review
├── 单品详情 /wardrobe/:id
├── 穿搭页面 /outfits
├── 穿搭结果 /outfits/:id
└── 设置 /settings
```

ChatGPT App 不复制完整管理网页，只提供高频操作组件：

- 单品识别确认卡。
- 衣橱摘要卡。
- 单品列表卡。
- 穿搭组合卡。
- 修改和删除确认卡。

## 2. 首次使用流程

```text
邀请链接
  → 校验邀请码
  → 查看产品说明与数据权限
  → 注册或登录
  → 连接 ChatGPT App
  → 授权成功
  → 打开预设对话
  → 引导添加第一件单品
```

状态与处理：

| 状态 | 页面行为 |
|---|---|
| 邀请有效 | 显示“建立我的衣橱”按钮 |
| 邀请过期 | 说明已过期，提供申请新邀请入口 |
| 邀请已达使用次数 | 不允许继续绑定 |
| 已登录未授权 | 继续 ChatGPT 连接流程 |
| 已登录已授权 | 直接打开衣橱或预设对话 |
| 授权被撤销 | 保留衣橱数据，引导重新连接 |

## 3. 添加单品流程

```text
选择图片
  → 客户端预检格式与大小
  → 上传临时私有图片
  → AI 分析
  → 展示候选字段
  → 用户确认或修改
  → 正式创建单品
```

用户可编辑字段仅包括：

- 名称。
- 大类。
- 二级分类。
- 主色。

AI 后台字段如风格、季节和置信度不作为必填表单展示。

失败状态：

| 情况 | 处理 |
|---|---|
| 文件格式不支持 | 上传前提示支持的格式 |
| 文件过大 | 上传前提示压缩或重拍 |
| 图片无法读取 | 不建立记录，允许重试 |
| 疑似多件单品 | 提示“一次只拍一件”，允许更换图片 |
| 不是穿搭单品 | 不保存，提示重新上传 |
| AI 暂时失败 | 保留临时图片，允许重试或手动填写 |
| 用户取消 | 删除或等待清理临时图片 |
| 重复确认 | 通过幂等键避免重复创建 |

## 4. 衣橱首页

默认展示：

- 全部单品数量。
- 上装、下装、鞋履、包袋数量。
- 分类筛选标签。
- 单品卡片网格。
- “添加单品”主按钮。
- 空衣橱引导。

单品卡片显示：

- 缩略图。
- 名称。
- 大类。
- 主色。

MVP 默认按创建时间倒序排列，不提供复杂排序和搜索。

## 5. 单品详情与修改

详情页显示原图、确认字段和创建时间。修改保存后立即反映到衣橱和后续穿搭推荐。

删除必须二次确认：

> 删除后，这件单品将不再出现在衣橱和新的穿搭推荐中。是否继续？

删除成功后返回衣橱首页；历史穿搭可保留文本快照，但不能把已删除单品作为当前可用项。

## 6. 穿搭流程

```text
输入自然语言需求
  → 生成中
  → 返回 1–3 套
  → 查看组合与解释
  → 可重新生成
```

结果卡包含：

- 拼图或可交互单品卡片组合。
- 单品名称列表。
- 一句搭配说明。
- “全部来自你的衣橱”标识。
- 若存在合理缺口，单独显示缺口说明。

特殊状态：

| 情况 | 结果 |
|---|---|
| 空衣橱 | 引导先添加单品 |
| 缺少一个必要类别 | 先展示能组成的部分，再说明缺口 |
| 单品足够 | 不显示购买建议 |
| 请求过于模糊 | 使用“简单日常”默认解释，不阻塞用户 |
| 生成失败 | 保留输入，允许重试 |

## 7. ChatGPT 对话流程

### 新增

1. 用户上传图片并要求加入衣橱。
2. ChatGPT 调用图片分析/新增准备工具。
3. App 显示确认卡。
4. 用户确认后调用正式新增工具。
5. 返回保存成功和当前分类数量。

### 查询

“我有哪些鞋？”调用筛选查询；“我的衣橱有什么？”调用摘要，不一次返回所有原图。

### 修改与删除

- 必须先定位唯一单品。
- 多个候选时让用户选择，不能猜测。
- 删除必须展示目标并确认。

### 推荐

1. ChatGPT 传递用户原始需求。
2. MCP 服务查询当前用户衣橱并生成结果。
3. App 展示结构化组合。
4. 模型只负责简短叙述，不得替换服务端返回的单品 ID。

## 8. 可用性验收

- 手机宽度 375px 时无横向溢出。
- 核心按钮具有明确文字，不只使用图标。
- 生成中、空状态、错误状态和成功状态均有明确反馈。
- 表单错误显示在对应字段附近。
- 删除等不可逆操作不能只靠自然语言推断执行。
- 新用户不阅读说明文档也能完成第一次录入。
