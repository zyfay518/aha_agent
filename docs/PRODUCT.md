# 产品设计：Aha Agent AI 线上衣橱

> 文档性质：产品目标与范围基线  
> 当前版本：0.1  
> 建立日期：2026-07-12  
> 状态：MVP 已定义，待实现

## 1. 产品愿景

帮助用户先穿好自己已经拥有的衣服，而不是因为“不知道穿什么”继续冲动购物。

Aha Agent 是运行在 ChatGPT 中的个人线上衣橱助手。用户通过拍摄单件衣物建立数字衣橱，随后可通过自然语言查询衣橱、管理单品，并让系统优先使用已有衣物生成简单协调的穿搭。

产品原则：

1. 先管理衣橱，再谈专业时尚。
2. 先使用已有单品，再建议购买。
3. 录入和使用必须足够简单，不要求用户学习服装专业知识。
4. MVP 先跑通最小闭环，复杂能力逐步增加。
5. 用户始终拥有其衣橱数据的查看、修改、删除和撤销授权能力。

## 2. 目标用户

第一批用户具有以下一个或多个特征：

- 已经拥有较多衣服，但经常觉得没有衣服可穿。
- 不擅长或不愿花时间研究穿搭。
- 容易重复购买相似单品。
- 希望减少冲动消费，提高已有衣物利用率。
- 希望通过 ChatGPT 对话完成衣橱管理和穿搭建议。

MVP 是个人工具，不包含造型师、好友协作或时尚社区。

## 3. 核心价值

用户应当能够快速回答三个问题：

1. 我的衣橱里已经有什么？
2. 我现有的衣服今天可以怎样搭？
3. 我是真的缺少某件单品，还是已经有相似的东西？

## 4. MVP 用户闭环

```text
收到 Aha Skill 与访问码
  → Skill 连接远程 MCP
  → 在 Agent 对话中发送一张单件衣物照片
  → 用户自己的 Agent 识别并由用户确认
  → 单品进入个人衣橱
  → 用户描述想要的穿搭
  → 系统从已有单品中生成 1–3 套搭配
  → 以单品卡片拼图展示
  → 只有确实无法合理搭配时才指出衣橱缺口
```

## 5. MVP 功能范围

### 5.1 用户与连接

- 用户可以注册、登录和退出。
- 用户通过邀请链接进入连接流程。
- 用户在 ChatGPT 中完成一次连接和授权后可长期使用。
- 用户可以主动撤销授权。
- 邀请码只用于邀请和建立连接，不是永久访问凭证。

受 ChatGPT 平台安全机制限制，MVP 目标是“一次确认连接”，不承诺仅发送普通文本码即可无确认自动安装第三方 App。

### 5.2 单品录入

- 每次上传一张图片。
- 一张图片只包含一件单品。
- 优先支持平铺图和衣架图。
- 支持 JPG、PNG、WebP。
- 用户使用的 Agent 直接识别：名称、大类、二级分类和颜色；Aha 后端不调用模型 API。
- 确认页以点击标签展示类别、季节和颜色三个维度；季节可多选，颜色最多双选。
- 二级分类和风格标签主要由 AI 或系统维护，不要求用户通过下拉框逐项填写。
- 用户确认或修改后才正式保存。
- 检测到多件单品时提示重新拍摄。

### 5.3 衣橱管理

- 查看全部单品。
- 按四大类筛选。
- 查看单品详情。
- 修改名称、类别、季节和颜色。
- 删除单品。
- 查看衣橱分类数量摘要。
- 通过移动端网页正常上传和管理。

主入口是支持 Skill/MCP 的 Agent 对话。独立网页仅作为备份录入和可选衣橱展示，不要求普通用户日常访问。

### 5.4 MVP 分类

| 大类 | 二级分类示例 |
|---|---|
| 上装 | T 恤、衬衫、针织衫、卫衣、外套、其他 |
| 下装 | 长裤、短裤、泳裤、半身裙、其他 |
| 鞋履 | 运动鞋、皮鞋、靴子、凉鞋、其他 |
| 包袋 | 手提包、单肩包、双肩包、其他 |

MVP 不包含首饰、帽子、眼镜等配饰，也不支持用户自定义分类。

### 5.5 穿搭推荐

- 用户可以用自然语言描述想要的穿搭。
- MVP 不采集身材、肤色、年龄等个体属性。
- MVP 不支持复杂穿衣禁忌或偏好设置。
- 一次生成 1–3 套搭配。
- 基础组合为上装、下装和鞋履；包袋可选。
- 推荐结果优先且尽可能只使用用户已有单品。
- 每套结果包含单品、拼图和一句简短说明。
- 结果不生成真人上身图。

### 5.6 减少冲动消费

推荐系统必须遵守以下顺序：

1. 查询用户已有单品。
2. 尝试生成完整的已有单品组合。
3. 只要存在基本协调的组合，就不提出购买建议。
4. 必要类别为空或现有单品确实无法完成需求时，才指出衣橱缺口。
5. 衣橱缺口只描述品类、颜色、版型等规格，不推荐具体商品或购物链接。
6. 当用户表达购买意图时，优先检查衣橱中是否存在相似单品。

### 5.7 ChatGPT 中的典型指令

- “把这件白色衬衫加入我的衣橱。”
- “我的衣橱里现在有什么？”
- “帮我搭一套简单舒服的穿搭。”
- “把刚才那件衣服改成下装。”
- “删除这双鞋。”
- “我想买黑色衬衫，看看我是不是已经有类似的。”

## 6. MVP 明确不做

- 真人或模特上身图。
- 身材、肤色和年龄分析。
- 一张图片自动拆分多个单品。
- 自定义分类和复杂标签系统。
- 洗涤、借出、淘汰等衣物状态。
- 首饰及复杂配饰。
- 衣橱分享、好友协作和造型师协作。
- 社交社区。
- 电商链接、比价、自动购买和商业化。
- 多 Agent 平台适配。
- 仅凭文本邀请码绕过用户确认自动安装。

## 7. MVP 总体验收标准

MVP 只有同时满足以下条件，才算完成：

1. 新用户安装 Skill、发送访问码后能够连接自己的衣橱。
2. 新用户能在 5 分钟内录入至少 10 件准备好的单品。
3. 单件录入包含对话图片、宿主 Agent 识别、用户确认和 MCP 保存的完整流程。
4. 两个账号的数据严格隔离。
5. 用户可以查看、分类、修改和删除自己的单品。
6. 衣橱中上装、下装和鞋履齐全时，系统能生成 1–3 套已有单品组合。
7. 推荐结果中的每个单品都真实存在于当前用户衣橱。
8. 系统不会在存在合理搭配时无故建议购买。
9. ChatGPT 可以完成查询、新增、修改、删除和穿搭推荐。
10. 用户在新的 ChatGPT 对话中仍可使用已连接的服务。
11. 用户可以撤销授权并删除账号。
12. 删除流程能按隐私规则清除数据库记录和图片。

## 8. MVP 后的升级顺序

1. 一张照片识别多个单品。
2. 上传灵感图并匹配用户衣橱。
3. 用户穿搭偏好和禁忌。
4. 天气、场合和日历信息。
5. 穿着次数和衣物状态。
6. 自定义分类与标签。
7. 旅行打包。
8. 更复杂的搭配质量评分。
9. 其他支持 MCP 的 Agent 平台。
10. 商品推荐和商业化。

## 9. 产品变更规则

- 本文档记录当前生效的产品定义，可以随确认后的需求更新。
- 每次范围变化必须先在 `PROJECT_LOG.md` 中追加变更记录，再更新本文档。
- 删除旧目标时不能直接抹除历史，应在日志中说明删除原因和影响。
- 新功能不得削弱“优先使用已有衣物、减少冲动消费”的产品主线。
